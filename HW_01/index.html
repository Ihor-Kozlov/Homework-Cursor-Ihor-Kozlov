Homework 1

Base

  • git config --global user.name blablabla - задає ім'я користувача
  • git config --global user.email blabla@blabla - задає електронну адресу користувача
  • git init - створює git-репозиторій
  • git add - додає наповнення нового коміту
  • git commit - робить коміт (переносить дані з index в репозиторій)
  • git log - перегляд історії комітів
  • git checkout - перехід між гілками, перегляд данних з гілки
  • git branch - ця команда показує наявні гілки. Може додавати, видаляти, або змінювати їх назву
  • git push - "мерж" локальних змін у віддалений репозиторій
  • git pull - "мерж" змін із віддаленого репозиторію в локальний. Також це комбінація двох команд: git fetch і git merge
  • git merge - злиття декількох гілок в одну
  • git clone - клонування віддаленого репозиторію
  • .gitignore - це файл, в який записується всі дані, які повинен ігнорувати git

Advanced

  • git revert - скасовує існуючий коміт
  • git reset - скасовує останній зроблений коміт
  • git rebase - ця команда об'єднює коміти однієї гілки з іншою
  • git cherry-pick - бере зміни, які вносяться один комітом, і повторно застосовує їх у вигляді нового коміту в данній гілці
  • git stash - дозволяє на певний час відкласти зміни. Можна "приховати" свої зміни. Ці зміни поміщаються в стек, тоді як ви можете переключитись на іншу гілку.
